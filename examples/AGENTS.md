# examples — VMOD gallery

See also: [varnish/](../AGENTS.md) · [backend subsystem](../varnish-sys/src/vcl/backend/AGENTS.md)

15 minimal VMODs + `stat_tree` binary, all workspace members. Each demonstrates one framework feature in isolation — read the matching one before implementing similar functionality elsewhere.

## Shared skeleton (every vmod_* crate)

- `Cargo.toml`: `crate-type = ["cdylib"]`, `varnish.workspace = true`, `publish = false`, `version = "0.0.0"`.
- `src/lib.rs`: `#[varnish::vmod(docs="README.md")] mod name { ... }` + `varnish::run_vtc_tests!("tests/*.vtc")`.
- `README.md`: auto-generated by the macro from doc comments, marked **DO NOT EDIT** — edit the doc comments in `lib.rs` instead.
- No `.vcc` files. This framework replaces the classic VCC toolchain entirely.

## Gallery

| example | demonstrates |
|---|---|
| `vmod_example` | basic tutorial vmod (`sum`, `reverse`) — companion to `varnish/src/lib.rs` docs |
| `vmod_object` | Rust struct exposed as VCL object (string dict), constructor + methods |
| `vmod_event` | `#[event]` init/fini/warm/cold lifecycle + `#[shared_per_vcl]` state |
| `vmod_subcall` | calling VCL subroutines from Rust (`Subroutine`, `call_sub`), per-task context |
| `vmod_timestamp` | `#[shared_per_task]` demo (duration since last call, same task) |
| `vmod_counters` (crate `stats`) | custom VSC metrics via `Vsc`/`#[derive(VscMetric)]` |
| `vmod_blobutils` | `VCL_BLOB`/`&[u8]` argument handling |
| `vmod_error` | error handling patterns (`Result<T, VclError/&str/String>`) |
| `vmod_infiniteloop` | ignoring restart/retry limits — VCL control-flow edge case |
| `vmod_restricted_callsites` | `#[restrict(...)]` — limits which VCL subs may call a fn, enforced at load time |
| `vmod_native_backend`, `vmod_simple_backend` | custom backend/director impls (`VclBackend`/`VclResponse`) |
| `vmod_round_robin` | round-robin director, load balancing across backends |
| `vmod_vdp` | Delivery Processor — mutates `resp.body` on way out to client |
| `vmod_vfp` | Fetch Processor — mutates body as received from backend |
| `stat_tree` | standalone `[[bin]]`, not a VMOD — uses `MetricsReader` to render VSC stats hierarchically by dot-separated section; own `tests/vtc.rs` |

## Backend examples

`vmod_native_backend`/`vmod_simple_backend`/`vmod_round_robin` implement traits from [`varnish-sys/src/vcl/backend/`](../varnish-sys/src/vcl/backend/AGENTS.md) — read that guide alongside these.
